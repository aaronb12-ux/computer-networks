# Connectionless Transport: UDP

**UDP (User Datagram Protocol)** is a minimal, connectionless transport protocol that provides **best-effort delivery**.

- No guarantees on delivery  
- Packets may be **lost** or **arrive out of order**  
- Often described as a **“no frills”** or **“bare bones”** protocol  

---

## Key Characteristics

### Connectionless
- No handshaking between sender and receiver  
- Each segment is treated **independently**  

### Lightweight
- Small header size  
- No connection state maintained  

### No Congestion Control
- Can send data at any rate  
- Continues operating even under network congestion  

---

## Why Use UDP?

- No connection setup → **lower latency (no RTT delay)**  
- Simple protocol design  
- Low overhead  
- Suitable for time-sensitive applications  

---

## Common Use Cases

- Streaming multimedia  
- DNS  
- SNMP  
- HTTP/3 (built on QUIC over UDP)  

---

## Reliability with UDP

UDP itself does **not** provide reliability.

If needed, it must be implemented at the **application layer**:
- Reliability (acknowledgments, retransmissions)  
- Congestion control  

---

## Transport Layer Actions

### Sender
- Receives application-layer message  
- Constructs UDP segment (adds header)  
- Passes segment to IP layer  

### Receiver
- Receives segment from IP layer  
- Verifies checksum  
- Extracts message  
- Delivers to application via socket  

---

## UDP Segment Header

UDP has only **4 fields**:

- Source port  
- Destination port  
- Length (includes header + data)  
- Checksum  

---

## UDP Checksum

**Purpose:** Detect bit errors during transmission  

### Sender
- Treats segment as sequence of 16-bit integers  
- Computes checksum using **one’s complement sum**  
- Stores result in checksum field  

### Receiver
- Recomputes checksum  
- Compares with received value:
  - Match → no error detected  
  - Mismatch → error detected  

> Note: Matching checksums do **not guarantee** no errors (rare collisions possible)

---

## Summary

UDP is a **fast, simple, connectionless protocol**:

- No setup or handshaking  
- No reliability guarantees  
- No ordering guarantees  
- Basic error detection only  

### Advantages
- Low latency  
- Minimal overhead  
- Works under congestion  

### Tradeoff
- Reliability and control must be handled by the application
