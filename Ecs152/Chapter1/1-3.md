## The Network Core

Mesh of interconnected routers.

**Packet-switching:** hosts break application-layer messages into packets
- The end host takes application level messages, divides those messages into chunks of data, and puts those chunks of data into packets and sends those packets into the internet
- The network then forwards packets from one router to the next, across links on path from source to destination

### Two Key Network-Core Functions:

### Forwarding:
**aka:** "switching"

**Local action:** move arriving packets from router's input link to appropriate router output link

- Forwarding is controlled by a forwarding table, and there is a forwarding table inside each of the millions of routers in the internet
- When the packet arrives, the router looks inside the packet for a destination address and then looks up the destination address in the forwarding table. It then transports that incoming packet on the output link that leads to the destination

**How are the contents of that forwarding table created in the first place?**

### Routing:
**Global action:** determine source-destination paths taken by packets

- Routing algorithms compute the forwarding tables

### Packet-Switching: Store-and-Forward

**Packet Transmission Delay:** takes L/R seconds to transmit L-bit packet into link at R bps

**Store and Forward:** entire packet must arrive at router before it can be transmitted on next link

### Packet-Switching: Queueing

If too many packets from different sources arrive to the hop-router at a rate that is much faster than the rate of the transmission over the output link, then a queue of packets will form in the first hop-router. Queueing occurs when work arrives faster than it can be serviced.

**Packet queueing and loss:** if arrival rate (in bps) to link exceeds transmission rate (bps) of link for some period of time:
- Packets will queue, waiting to be transmitted on output link
- Packets can be dropped (lost) if memory (buffer) fills up in router

### Internet Structure: A "Network of Networks"

- Hosts connect to internet via access Internet Service Providers (ISPs)
- Access ISPs in turn must be interconnected, so that any two hosts can send packets to each other

**Given millions of access ISPs, how to connect them together?**

❌ **Option 1:** If we wire each access ISP to every other ISP, this would be O(N²) connections → it won't scale, won't work

✓ **Option 2:** Connect each access ISP to one global transit ISP? Customer and provider ISPs have economic agreement

✓ **Option 3:** Multiple competing global ISPs with regional networks and Internet Exchange Points (IXPs)

**Modern Internet Structure:**
- At the center: small number of well-connected large networks
  - Tier-1 commercial ISPs (e.g., Level 3, Sprint, AT&T, NTT)
  - Content provider networks (e.g., Google, Facebook, Microsoft, Akamai)
- Access ISPs and regional ISPs connect to these core networks
- IXPs (Internet Exchange Points) allow networks to peer directly
