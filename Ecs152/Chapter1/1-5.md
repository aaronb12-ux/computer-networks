## Layering and Encapsulation

### What will we be talking about?

Layers, Service, Encapsulation: architectural layering, internet layers, encapsulation

**Layers:** each layer implements a service
- Via its own internal-layer actions
- Relying on services provided by layer below

### Why Layering?

**Approach to designing/discussing complex systems:**
- Explicit structure allows identification, relationship of system's pieces
- Layered reference model for discussion

**Modularization eases maintenance, updating of system:**
- Change in layer's service implementation: transparent to rest of system
- Change in gate procedure doesn't affect rest of system

### Layered Internet Protocol Stack

**Application:** supporting network applications
- HTTP, IMAP, SMTP, DNS

**Transport:** process-process data transfer. Transports application layer messages from one process to another.
- TCP, UDP

**Network:** routing of datagrams from source to destination. Transports data from one end device/host to another.
- IP, routing protocols

**Link:** data transfer between neighboring network elements
- Ethernet, 802.11 (WiFi), PPP

**Physical:** bits "on the wire"

### Services, Layering and Encapsulation

**Encapsulation:** taking a data unit from a higher layer, adding info to create a new data unit at another layer is called encapsulation.

**Application Layer:**
- Application exchanges messages (M) to implement some application service using services of transport layer
- Protocol data unit is known as a **message**

**Transport Layer:**
- Transport-layer protocol transfers M (e.g., reliably) from one process to another, using services of network layer
- Transport-layer protocol encapsulates application-layer message M with transport layer header H<sub>t</sub> to create a transport-layer **segment**
- H<sub>t</sub> used by transport layer protocol to implement its service

**Network Layer:**
- Network-layer protocol transfers transport-layer segment {H<sub>t</sub> | M} from one host to another, using link layer services
- Network-layer protocol encapsulates transport-layer segment {H<sub>t</sub> | M} with network layer header H<sub>n</sub> to create a network-layer **datagram**
- H<sub>n</sub> used by network layer protocol to implement its service
- **Datagram:** protocol data unit that is used at the network layer

**Link Layer:**
- Link-layer protocol transfers datagram {H<sub>n</sub> | {H<sub>t</sub> | M}} from host to neighboring host, using network-layer services
- Link-layer protocol encapsulates network datagram {H<sub>n</sub> | {H<sub>t</sub> | M}} with link-layer header H<sub>l</sub> to create a link-layer **frame**
