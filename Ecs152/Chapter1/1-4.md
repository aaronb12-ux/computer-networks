How do packet delay and loss occur?  
- packets queue in router buffers, waiting for their turn for transmission  
  queue length grows when arrival rate to link temporarily exceeds output link capacity  

- packet loss occurs when memory to hold queued packets fills up

Processing Delay:
- delay associated with forwarding table lookup, forwarding a packet through the switch, and doing integrity tests  
- dproc: nodal processing  
  - check bit errors  
  - determine output link  
  - typically < microseconds  

---

Queueing Delay:
- amount of time a packet needs to wait in queue at an output link for transmission  
- depends on congestion level of router  

---

Transmission Delay:
- once a packet is being sent through the link at transmission rate R, it takes time for all the bits in the packet to be pushed into the outgoing link → transmission delay  

- L: packet length (bits)  
- R: link transmission rate (bps)  

- dtrans = L / R  

---

Propagation Delay:
- time it takes for a bit, once it enters the sending side of the link, until it reaches the receiving side  

- d: length of physical link  
- s: propagation speed  

- dprop = d / s  

---

"Real" Internet Delays and Routes

traceroute program: provides delay measurements from source to router along the end-to-end internet path toward destination. For all i:  
- sends three packets that will reach router i on the path toward destination (with time-to-live field value of i)  
- router i will return packets to sender  
- sender measures time interval between transmission and reply  

How it works:  
- first sends 3 packets to the first hop router, the first hop router sends a reply message in response to each of these packets  
- the traceroute sender measures the Round Trip Time (RTT) from when it sends the message to when it gets the reply and then displays those 3 RTT measurements  
- it then sends 3 packets to the second hop router, measures and displays the RTT, and continues until the final destination is reached

Throughput:
- rate (bits/second) at which bits are being sent from sender to receiver  
  - instantaneous: rate at a given point in time  
  - average: rate over a longer period of time  

- a sender sends bits through a path. each transmission link across a path is able to send those bits at some rate Rs bits/sec  

- when you think of the throughput that a source to destination path can receive, it is limited by the capacity of the thinnest pipe

Throughput: Network Scenario

- need to think about how individual flows interact with each other in terms of throughput  

- we have 10 servers and 10 clients, and a single connection from each server to one of the clients  
- in the center of the network, we have a shared link of capacity R  
- this shared link fairly shares bandwidth among the 10 flows so that each sees bandwidth of R/10  

What is the per-connection throughput in this scenario?

- each session passes through 3 pipes: Rs, R/10, and Rc  

- per-connection end-to-end throughput = min(Rs, R/10, Rc)  

---

In practice:
- Rc or Rs is smaller than R/N  
- bottleneck link is at network edge
