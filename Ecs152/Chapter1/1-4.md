## Review of Last Section

### How Do Packet Delay and Loss Occur?

- Packets **queue in router buffers**, waiting their turn for transmission  
- Queue length grows when the **arrival rate temporarily exceeds the output link capacity**

- **Packet loss** occurs when the buffer (memory) used to hold queued packets becomes full

### 1. Processing Delay

- Time required to:
  - Check for bit errors
  - Determine the outgoing link (routing decision)
- Denoted as **d_proc**
- Typically very small (**microseconds**)

---

### 2. Queueing Delay

- Time a packet waits in the output buffer before transmission
- Depends on the **level of congestion** at the router
- Can vary significantly

---

### 3. Transmission Delay

- Time required to push all packet bits onto the link

- **L** = packet length (bits)  
- **R** = transmission rate (bits/sec)

- **d_trans = L / R**

---

### 4. Propagation Delay

- Time for a bit to travel across the physical link

- **d** = length of the link  
- **s** = propagation speed

- **d_prop = d / s**

## "Real" Internet Delays and Routes

### Traceroute

The **traceroute** program measures delay from the source to each router along the end-to-end path.

For each hop *i*:
- Sends **3 packets** with TTL = *i*
- Router *i* returns a reply
- Sender measures the **round-trip time (RTT)**

### How It Works

- Sends 3 packets to the **first-hop router**
- That router responds → RTT is measured
- Repeats for the **second hop**, third hop, etc.
- Continues until reaching the **destination**

## Throughput

- **Throughput** = rate (bits/sec) at which data is delivered from sender to receiver

### Types
- **Instantaneous throughput** → rate at a specific moment  
- **Average throughput** → rate over time  

---

### Key Idea

A path consists of multiple links, each with its own rate.

- The **end-to-end throughput is limited by the bottleneck link**
- Think of it like **fluid flowing through pipes**:
  - The **narrowest pipe limits the flow**

## Throughput: Network Scenario

- 10 servers and 10 clients  
- Each server communicates with one client  
- A **shared link** has capacity **R**

- The shared link divides bandwidth fairly:
  - Each flow gets **R / 10**

### Per-Connection Throughput

Each connection passes through:
- **Rs** (server link)
- **R / 10** (shared link)
- **Rc** (client link)

**End-to-end throughput:**

min(Rs, R/10, Rc)

---

### Practical Insight

- Often, **Rs or Rc < R / N**
- So the **bottleneck is usually at the network edge**
